# Jump Space 战利品分配规则

## 1. 权重系统

### m_WeightXY (Vector2)

每个 `LootTypeAmount` 的 `m_WeightXY` 字段定义了该物品类型的权重范围：
- **x** = CR1 时的权重
- **y** = CR5 时的权重
- CR2~CR4 的权重由 `ChallengeRatingFloat.Get(float cr)` **线性插值**计算

```
权重(cr) = x + (y - x) × (cr - 1) / 4
```

### ChallengeRatingFloat

`GoPickerNode_ChallengeRatedAssetRef` 中的 `ChallengeRatingFloat` 存储5个独立权重值：
- `m_CR1` ~ `m_CR5`：分别对应5个挑战等级的权重
- `Get(float cr)`：根据当前CR线性插值返回权重
- `Value` 属性：返回当前CR对应的插值结果

---

## 2. 权重累加范围

**同一 `LootBundle` 内**所有 `LootTypeAmount` 的权重累加为 `weightSum`。

```
weightSum = Σ(每个LootTypeAmount的权重)
```

- CR1 时：`minWeightSum = Σ(m_WeightXY.x)`
- CR5 时：`maxWeightSum = Σ(m_WeightXY.y)`

---

## 3. 概率计算

单个物品的选中概率 = 该物品权重 / 同Bundle权重总和

```
概率 = 单项权重 / weightSum
```

- CR1 概率：`probCR1 = wMin / minWeightSum`
- CR5 概率：`probCR5 = wMax / maxWeightSum`

> **注意**：概率是权重的**比值**，不是权重本身。权重是绝对数值，概率是相对比例。

---

## 4. 固定 vs 随机

`LootTypeAmount.m_Amount` 决定分配方式：

| m_Amount | 标记 | 含义 |
|----------|------|------|
| > 0 | ◆ 固定 | 该物品**必定**出现指定数量，不参与权重分配 |
| = 0 | ◇ 随机 | 该物品按权重概率参与随机分配 |

固定物品先从总件数中扣除，剩余件数再按权重分配给随机物品。

---

## 5. AllocateLargestRemainder（最大余额法）

**用途**：将 `m_AmountOfLootInGroup`（总件数）按权重比例**整数分配**给各随机类型，确保总数精确。

**签名**：`int[] AllocateLargestRemainder(int total, float[] weights, float weightSum, LootBundle bundle)`

### 算法步骤

1. **计算应得件数** = 总件数 × (权重 / 权重总和)
2. **取整数部分**（floor），先分配这些
3. **统计已分配件数**，计算差额 = 总件数 - 已分配
4. **按小数部分（余额）从大到小排序**，差额个类型各补1

### 示例

总件数=4，权重：A=0.50, B=0.30, C=0.20

| 类型 | 应得 | 整数部分 | 余额 | 最终 |
|------|------|----------|------|------|
| A | 4×0.50=2.0 | 2 | 0.0 | 2 |
| B | 4×0.30=1.2 | 1 | 0.2 | 1 |
| C | 4×0.20=0.8 | 0 | 0.8 | 1 |

已分配3件，差1件 → C余额最大补1 → **A=2, B=1, C=1**

### 核心保证

分配结果的总和**精确等于** `m_AmountOfLootInGroup`，不会因取整而多一件或少一件。

### 调用点

| 调用方法 | 场景 | 调用次数 |
|----------|------|----------|
| `GetLootTypeAmountFromBundle()` | 运行时主方法 | 1次 |
| `SetMinAndMaxAmountsForAllLootAmountsInBundle()` | 编辑器预览 | 4次(CR1~CR5) |
| `EvaluateAllocationForIndex()` | 辅助方法 | 内部调用 |

## 6. PickIndexRoulette（轮盘赌法）

**用途**：逐件按权重概率随机选取物品类型（返回单个索引）。

**签名**：`int PickIndexRoulette(float[] weights, float totalWeight)`

### 算法步骤

1. 生成随机数 `r ∈ [0, totalWeight)`
2. 从第一个权重开始累加，跳过权重为0的项
3. 当累加值 ≥ r 时，返回当前索引
4. 若遍历完未命中，从末尾向前回溯找第一个非零权重
5. 全部权重为0时返回 -1

### ⚠️ 实际状态：未使用（死代码）

经ISIL反编译全量调用搜索确认：**`PickIndexRoulette` 在整个代码库中无任何调用点**，仅存在方法定义。

游戏运行时**只使用 `AllocateLargestRemainder`（最大余额法）**进行战利品分配。

### 与最大余额法的区别

| 特性 | 最大余额法 | 轮盘赌法 |
|------|-----------|---------|
| 返回值 | `int[]`（每种类型的件数） | `int`（单个选中索引） |
| 确定性 | 确定性分配（同权重同结果） | 随机性分配（每次结果不同） |
| 总数保证 | 精确保证 | 统计上保证（大数定律） |
| 调用方式 | 一次调用分配全部件数 | 需循环调用N次选N件 |
| 零权重处理 | 自然跳过（应得=0） | 显式跳过+回溯 |
| 实际使用 | ✅ 唯一使用的分配算法 | ❌ 未使用（死代码） |

## 7. 数据结构层级

```
LootManagerData (活动战利品配置)
 └─ LootBundleContainer[] (容器组)
     └─ LootBundle[] (战利品包)
         └─ LootTypeAmount[] (物品类型+数量+权重)
             ├─ m_Type: LootType (物品类型枚举)
             ├─ m_Amount: int (固定数量，0=按权重随机)
             └─ m_WeightXY: Vector2 (CR1权重, CR5权重)
```
